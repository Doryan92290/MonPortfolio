Section 01

Introduction — FenziConnect & enjeux

Présentation de l'entreprise

FenziConnect est une entreprise éditrice de solutions logicielles spécialisée dans la gestion d'entreprise (ERP/CRM). Elle accompagne ses clients dans la digitalisation de leurs processus administratifs, commerciaux et opérationnels au travers de sa plateforme centralisée CoreFlow.

La structure propose des modules couvrant la gestion des ressources humaines, le suivi client (CRM), la facturation et le pilotage de projets internes. L'équipe de développement travaille en méthode Agile (Scrum) avec des sprints de deux semaines.

Enjeux de la digitalisation administrative

Dans de nombreuses structures, la gestion des factures reste un processus manuel, source d'erreurs de calcul, de pertes de documents et de délais rallongés. La transformation numérique de ces workflows répond à trois enjeux majeurs :

  • Fiabilité — Automatiser les calculs (HT, TVA, TTC) pour éliminer les erreurs humaines.
  • Traçabilité — Centraliser l'historique des factures et leurs états (EN_ATTENTE, PAYÉE, ANNULÉE).
  • Collaboration — Permettre à plusieurs collaborateurs d'accéder et d'agir sur les données en temps réel.

Mission confiée

J'ai intégré l'équipe de développement de FenziConnect en tant que stagiaire développeur fullstack. Ma mission principale consistait à concevoir et développer deux modules pour la plateforme CoreFlow :

  • Module de Facturation — API REST Spring Boot + interface Angular pour la gestion complète du cycle de vie des factures.
  • Module de Gestion des Tâches — Système d'affectation et de suivi des tâches pour les collaborateurs et stagiaires.

Section 02

Mission 1 — Gestion des Factures

🔧 Backend — Spring Boot

Le backend repose sur une API REST développée avec Spring Boot (Java 17), structurée selon une architecture en couches stricte pour respecter le principe de séparation des responsabilités.

Architecture N-tiers
Controller (HTTP) └── Service (Logique métier) └── Repository (JPA / PostgreSQL) └── Entités : Facture, LigneFacture, Client, Collaborateur

Endpoints CRUD développés :

  • POST /api/factures — Créer une nouvelle facture avec ses lignes de produits.
  • GET /api/factures — Récupérer toutes les factures (avec filtres par statut, client, date).
  • GET /api/factures/{id} — Récupérer le détail d'une facture.
  • PUT /api/factures/{id}/statut — Changer le statut d'une facture.
  • DELETE /api/factures/{id} — Supprimer une facture (avec contrôle métier).

Algorithme de calcul automatique :

Service — Calcul HT / TVA / TTC (Java)
public FactureDTO calculerMontants(List<LigneFacture> lignes) { BigDecimal montantHT = lignes.stream() .map(l -> l.getPrixUnitaire().multiply(BigDecimal.valueOf(l.getQuantite()))) .reduce(BigDecimal.ZERO, BigDecimal::add); BigDecimal tva = montantHT.multiply(new BigDecimal("0.20")); BigDecimal ttc = montantHT.add(tva); return new FactureDTO(montantHT, tva, ttc); }

Machine à états — Gestion des statuts :

  • EN_ATTENTE — État initial après création.
  • PAYÉE — Transition validée par le service comptabilité.
  • ANNULÉE — Annulation possible uniquement depuis EN_ATTENTE.

La validation des données est assurée par Bean Validation (@NotNull, @Positive, @Size) sur les DTOs, côté contrôleur, avant toute persistance en base.

🎨 Frontend — Angular

L'interface utilisateur a été développée avec Angular 16 (TypeScript) et SCSS. Elle se compose d'un dashboard liste et d'un formulaire de création avancé.

Dashboard — Liste des factures :

  • Tableau dynamique avec filtrage temps réel par statut via ngModel et pipe personnalisé.
  • Tri des colonnes (date, montant TTC, client, statut).
  • Codes couleurs SCSS par statut : vert (#2ECC71) payée, rouge (#E23636) annulée, orange (#F39C12) en attente.
  • Pagination côté client pour les listes longues.

Formulaire de création — FormArray dynamique :

Gestion dynamique des lignes produits (TypeScript)
// Initialisation du FormArray lignesArray = this.fb.array([this.creerLigneFormGroup()]); creerLigneFormGroup(): FormGroup { return this.fb.group({ designation: ['', Validators.required], quantite: [1, [Validators.required, Validators.min(1)]], prixUnitaire: [0, [Validators.required, Validators.min(0.01)]] }); } // Ajout d'une ligne à la volée ajouterLigne() { this.lignesArray.push(this.creerLigneFormGroup()); } // Calcul TTC temps réel get montantTTC(): number { return this.lignesArray.controls.reduce((acc, ctrl) => { const ht = ctrl.value.quantite * ctrl.value.prixUnitaire; return acc + ht * 1.20; }, 0); }
✓ Résultat

L'interface permet la création de factures multi-lignes avec calcul automatique du TTC en temps réel, sans aucun rechargement de page. Les formulaires intègrent une validation réactive complète avec messages d'erreur contextuels.

Section 03

Mission 2 — Gestion des Tâches

Méthodologie Agile — Scrum

FenziConnect applique la méthode Scrum avec des sprints de deux semaines. J'ai participé activement à l'ensemble du cycle Agile :

  • Daily Stand-up — Réunion quotidienne de 15 minutes : état d'avancement, blocages, prévisions.
  • Sprint Planning — Sélection des tickets Jira et estimation en story points.
  • Sprint Review — Démonstration des fonctionnalités développées au Product Owner.
  • Rétrospective — Analyse des points positifs et axes d'amélioration du sprint.

Module de Suivi des Tâches

En complément du module de facturation, j'ai développé un système de gestion de tâches interne pour les collaborateurs de FenziConnect.

Fonctionnalités :

  • Création et affectation de tâches aux collaborateurs, stagiaires et clients.
  • Cycle de vie des tâches : À FAIRE → EN COURS → TERMINÉE / BLOQUÉE.
  • Tableau Kanban intégré à l'interface CoreFlow avec drag-and-drop des tickets.
  • Filtrage par collaborateur, statut et date d'échéance.

Angular n'ayant pas été couvert en profondeur lors des cours BTS, j'ai dû acquérir cette compétence en autonomie directement sur le projet de stage. L'utilisation de l'IA (ChatGPT, Claude) a joué un rôle significatif dans ce processus d'apprentissage accéléré :

Cette approche illustre une compétence clé du développeur moderne : la capacité à s'auto-former en situation professionnelle, en utilisant les outils disponibles de manière efficace et critique.

Section 05

Conclusion & Perspectives

Bilan du stage

Ce stage chez FenziConnect a constitué une immersion complète dans le développement fullstack professionnel. Les apports principaux ont été :

  • Autonomie technique — Prise en main d'Angular en situation réelle, sans encadrement direct sur la techno. Capacité à livrer des fonctionnalités testées et déployées en production.
  • Rigueur de développement — Application systématique des bonnes pratiques : tests unitaires, validation des données, gestion des erreurs, documentation API Postman.
  • Travail en équipe — Participation active au workflow Git (branches, merge requests, code review) et au cycle Agile Scrum sur le projet CoreFlow.
  • Vision métier — Compréhension des enjeux réels d'un module de facturation : calcul, traçabilité, états, sécurité des données.

Difficultés rencontrées

  • Gestion des CORS — Configuration des en-têtes Spring Boot pour autoriser les requêtes depuis le frontend Angular en développement.
  • FormArray Angular — La gestion dynamique de formulaires imbriqués a nécessité une montée en compétence sur les Reactive Forms.
  • Respect des sprints — Adapter la charge de travail aux délais de livraison en environnement Agile avec une pression réelle.

Perspectives professionnelles

Ce stage confirme mon orientation vers le développement fullstack enterprise, avec un intérêt particulier pour les architectures backend robustes (Spring Boot, API REST) et les interfaces modernes (Angular, TypeScript).

L'expérience acquise sur la stack Angular / Spring Boot / PostgreSQL, utilisée dans de nombreux projets ERP et SaaS B2B, représente un atout direct pour les entretiens d'alternance et les premiers postes en développement.
